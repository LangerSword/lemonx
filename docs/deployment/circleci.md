# CircleCI Integration

lemon.test integrates with CircleCI through two modes: Machine Runner (recommended) and Webhook (legacy).

## Machine Runner Mode

In this mode, CircleCI assigns jobs directly to your machine runner. No webhooks, no external servers.

### CircleCI Config Structure

The config generated by `npx lemonx init` defines three jobs:

#### ai-test-loop

The full generate → run → fix cycle:

```yaml
ai-test-loop:
  machine: true
  resource_class: <namespace>/<resource-class>
  steps:
    - checkout
    - run:
        command: |
          RESPONSE=$(npx tsx /home/circleci/lemon/src/index.ts 2>&1)
          echo "$RESPONSE"
          # Check for failures
          if echo "$RESPONSE" | grep -q "⚠️.*Max iterations reached"; then
            exit 1
          fi
          if echo "$RESPONSE" | grep -q "❌"; then
            exit 1
          fi
```

**Behavior**:
- Clones your repo (CircleCI `checkout` step)
- Runs `src/index.ts` which discovers files, generates tests, and runs the fix loop
- Exits with code 1 if max iterations reached or errors occurred
- Exits with code 0 if all tests pass

#### ai-generate-tests

Generates tests only (no execution or fixing):

```yaml
ai-generate-tests:
  machine: true
  resource_class: <namespace>/<resource-class>
  steps:
    - checkout
    - run:
        command: npx tsx /home/circleci/lemon/src/index.ts --generate-only
```

#### ai-run-tests

Runs existing tests only:

```yaml
ai-run-tests:
  machine: true
  resource_class: <namespace>/<resource-class>
  steps:
    - checkout
    - run:
        command: npx vitest run --reporter=verbose
```

### Workflow Configuration

The default workflow runs on all branches except `main`:

```yaml
workflows:
  ai-test-and-fix:
    jobs:
      - ai-test-loop:
          filters:
            branches:
              ignore: main
```

Customize the branch filter as needed:

```yaml
# Run only on specific branches
filters:
  branches:
    only: [develop, staging]

# Run on all branches including main
# (remove the filters entirely)
```

### Pipeline Context Variables

The runner has access to these CircleCI environment variables:

| Variable | Description |
|---|---|
| `CIRCLE_BRANCH` | Current branch name |
| `CIRCLE_SHA1` | Current commit SHA |
| `CIRCLE_PROJECT_USERNAME` | GitHub username/org |
| `CIRCLE_PROJECT_REPONAME` | Repository name |
| `CIRCLE_WORKING_DIRECTORY` | Working directory (~/project) |

## Webhook Mode (Legacy)

In webhook mode, CircleCI sends a webhook to the lemon.test Express server, which clones the repo and runs the test-fix loop.

### CircleCI Config for Webhook Mode

```yaml
jobs:
  ai-test-loop:
    executor: node-executor
    steps:
      - checkout
      - run:
          command: |
            PAYLOAD=$(jq -n \
              --arg repo "$CIRCLE_PROJECT_USERNAME/$CIRCLE_PROJECT_REPONAME" \
              --arg branch "$CIRCLE_BRANCH" \
              --arg commit "$CIRCLE_SHA1" \
              '{repoUrl: $repo, branch: $branch, commitSha: $commit}')
            
            curl -s -X POST "$LEMON_WEBHOOK_URL/webhook/test-and-fix" \
              -H "Content-Type: application/json" \
              -H "X-Webhook-Signature: $LEMON_WEBHOOK_SECRET" \
              -d "$PAYLOAD"
```

### Required Environment Variables in CircleCI

| Variable | Description |
|---|---|
| `LEMON_WEBHOOK_URL` | URL of the lemon.test webhook server |
| `LEMON_WEBHOOK_SECRET` | HMAC-SHA256 secret for signature verification |

### Webhook Endpoints

| Endpoint | Method | Purpose |
|---|---|---|
| `/webhook/test-and-fix` | POST | Full generate + run + fix loop |
| `/webhook/generate-tests` | POST | Generate unit tests only |
| `/webhook/generate-integration-tests` | POST | Generate integration tests only |
| `/webhook/generate-e2e-tests` | POST | Generate E2E tests only |
| `/webhook/run-tests` | POST | Run existing tests only |
| `/health` | GET | Health check |

## Comparing Modes

| Aspect | Machine Runner | Webhook |
|---|---|---|
| **Infrastructure** | Your machine | Your server + CircleCI executor |
| **Code Location** | Never leaves your machine | Cloned to temp workspace |
| **Setup Complexity** | Low | Medium |
| **Network Requirements** | Outbound to CircleCI only | Inbound webhook access |
| **PR Creation** | Manual | Automatic (if configured) |
| **Recommended For** | Most teams | Teams with existing webhook infrastructure |
